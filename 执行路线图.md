## Phase 1 — 诊断与信息收集（预估 1-2 天）

**目标：** 确认 mYjJYFSJwydN.exe 是否真的调用 FBrowserVIP.dll 做验证

| 步骤 | 操作 | 工具 | 成功标准 |
|------|------|------|---------|
| 1.1 | 启动主程序，ProcMon 过滤进程名，监控 DLL 加载 | ProcMon | 确认 FBrowserVIP.dll 被加载 |
| 1.2 | 筛选 CreateFile / RegCreateKey 事件 | ProcMon | 发现配置文件/注册表读写 |
| 1.3 | 启动主程序，Wireshark 过滤 HTTP 流量 | Wireshark | 捕获验证请求 URL 与协议 |
| 1.4 | 用 Detect It Easy 确认PE结构 | DIE | 确认 DLL 无加密 |
| 1.5 | 记录所有输出，写入 debug_log.md | - | Node 1 记录就绪 |

**交付物：** `debug_log.md` → Node 1
**Git：** `git commit -m "Phase 1 诊断报告"` → `git tag v1.0-diagnosis`

## Phase 2 — FBrowserVIP.dll 逆向分析（预估 2-3 天）

**目标：** 弄清楚验证 API 的输入输出格式

| 步骤 | 操作 | 成功标准 |
|------|------|---------|
| 2.1 | IDA/Ghidra 打开 FBrowserVIP.dll，反编译关键函数 | 看懂 SetKey/GetError 伪代码 |
| 2.2 | 分析 `FBroOnlineLicenseControl_SetKey` 参数格式 | 确定 key 是字符串/hex/结构体 |
| 2.3 | 分析 `FBroOnlineLicenseControl_GetError` 返回值 | 确定 0=成功，其他错误码含义 |
| 2.4 | 分析 `FBroBrowser_IsLicenceKey` 内部逻辑 | 确定它是检查本地 flag 还是发请求 |
| 2.5 | 分析 `FBroOnlineLicenseControl_GetShowLicenseType` | 确定返回值（2=正式版，1=试用等） |
| 2.6 | 确定 DLL 内部是否有网络请求（curl/winhttp） | 确认验证是本地还是服务端 |

**交付物：** 关键函数签名表 + 返回值含义
**Git：** `git commit` → `git tag v2.0-reverse`

## Phase 3 — 代理 DLL 开发（预估 2-3 天）

**目标：** 编译 FBrowserVIP.dll 代理 DLL，拦截验证 API

| 步骤 | 操作 | 成功标准 |
|------|------|---------|
| 3.1 | 用 `gendef` 或手动列出全部 131 个导出函数 → `.def` 文件 | `.def` 文件完整 |
| 3.2 | 编写 `dll_main.c`：DllMain + 加载原始 DLL | 编译通过 |
| 3.3 | 实现 6 个验证 API 拦截函数（返回固定成功值） | 编译通过 |
| 3.4 | 实现 125 个转发函数（.def 转发器或用宏生成） | 编译通过 |
| 3.5 | VS 2022 cl.exe 编译为 x86 DLL | `FBrowserVIP.dll` 产出 |
| 3.6 | 备份原始 DLL → 放入代理 DLL | 准备就绪 |

**交付物：** `src/FBrowserVIP_proxy.c` + `src/fbro_export.def` + `src/compile.bat`
**Git：** `git commit` → `git tag v3.0-dll-proxy`

## Phase 4 — 测试与迭代优化（预估 1-2 天）

**目标：** 绕过验证，程序稳定运行

| 步骤 | 操作 | 结果处理 |
|------|------|---------|
| 4.1 | 放入代理 DLL，启动主程序 | ✅ 成功 → 直接完成 |
| 4.2 | 验证被绕过，但有功能异常 | ⚠️ 逆向更多函数补齐返回值 |
| 4.3 | 程序崩溃/无法启动 | ⚠️ ProcMon 查哪个导出函数导致 |
| 4.4 | DLL 劫持本身失败（VMP 检测） | ⚠️ 切换到 IAT Hook 方案 |

**交付物：** 稳定运行的代理 DLL + `debug_log.md` → Node 4
**Git：** `git commit` → `git tag v4.0-stable`

## Phase 5 — 备选方案（仅 Phase 3+4 失败时启动）

| 方案 | 触发条件 | 难度 | 预估时间 |
